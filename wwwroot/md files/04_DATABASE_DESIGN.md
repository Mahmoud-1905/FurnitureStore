# 04 — Database Design
> **Destination:** Antigravity  
> **Purpose:** Converts the idea into tables with columns, types, constraints, and relationships.  
> **Rule:** Every FK becomes a Navigation Property in the EF Core model. Do not confuse ApplicationUser with a separate Users table — Identity manages the user table (AspNetUsers).

---

## Database Configuration
- **Production:** SQL Server — connection string in `appsettings.json`
- **Development:** SQLite — connection string in `appsettings.Development.json`
- **ORM:** Entity Framework Core (Code-First with Migrations)
- **Context class:** `AppDbContext : DbContext`
- **Identity:** `ApplicationUser : IdentityUser` — extends Identity's AspNetUsers table

---

## Table 1 — ApplicationUser (extends IdentityUser)
> Managed by ASP.NET Core Identity. Stored in AspNetUsers.

| Column | Type | Constraints | Notes |
|---|---|---|---|
| Id | nvarchar(450) | PK | GUID — auto-generated by Identity |
| FullName | nvarchar(150) | NOT NULL | Display name |
| Email | nvarchar(256) | UNIQUE, NOT NULL | Login credential |
| PasswordHash | nvarchar(MAX) | NOT NULL | PBKDF2 hash via Identity |
| PhoneNumber | nvarchar(20) | NULL | Optional |
| IsActive | bit | NOT NULL, DEFAULT 1 | 0 = deactivated by Admin; login blocked |
| CreatedAt | datetime2 | NOT NULL, DEFAULT GETUTCDATE() | Account creation timestamp |

**Roles (stored in AspNetRoles via Identity):**
- `Customer` (default on registration)
- `StoreManager`
- `Admin`

## Entity Relationships
| From | Relationship | To | Cardinality |
|---|---|---|---|
| ApplicationUser | places | Orders | 1:N |
| ApplicationUser | has one | Cart | 1:1 (UNIQUE) |
| ApplicationUser | writes | Reviews | 1:N |
| ApplicationUser | books | Appointments | 1:N |
| Categories | contains | Products | 1:N |
| Products | has | ProductImages | 1:N (CASCADE) |
| Products | included in | OrderItems | 1:N (NO CASCADE) |
| Products | in | CartItems | 1:N |
| Products | has | Reviews | 1:N (CASCADE) |
| Orders | contains | OrderItems | 1:N (CASCADE) |
| Carts | contains | CartItems | 1:N (CASCADE) |

## ER Diagram (dbdiagram.io syntax)
```
Table Users {
  Id nvarchar(450) [pk]
  FullName nvarchar(150) [not null]
  Email nvarchar(256) [unique, not null]
  IsActive bit [default: 1]
}
Table Categories {
  CategoryId int [pk, increment]
  Name nvarchar(100) [unique, not null]
  IsActive bit [default: 1]
}
Table Products {
  ProductId int [pk, increment]
  CategoryId int [ref: > Categories.CategoryId]
  Name nvarchar(200) [not null]
  Price decimal(10,2) [not null]
  StockQuantity int [not null, default: 0]
  SKU nvarchar(50) [unique, not null]
  IsActive bit [default: 1]
}
Table ProductImages {
  ImageId int [pk, increment]
  ProductId int [ref: > Products.ProductId]
  ImageUrl nvarchar(500) [not null]
  IsPrimary bit [default: 0]
}
Table Orders {
  OrderId int [pk, increment]
  UserId nvarchar(450) [ref: > Users.Id]
  Status nvarchar(30) [default: 'Pending']
  TotalAmount decimal(10,2) [not null]
  DeliveryAddress nvarchar(600) [not null]
  PlacedAt datetime2 [not null]
}
Table OrderItems {
  OrderItemId int [pk, increment]
  OrderId int [ref: > Orders.OrderId]
  ProductId int [ref: > Products.ProductId]
  Quantity int [not null]
  UnitPrice decimal(10,2) [not null]
  Subtotal decimal(10,2) [not null]
}
Table Carts {
  CartId int [pk, increment]
  UserId nvarchar(450) [ref: - Users.Id]
}
Table CartItems {
  CartItemId int [pk, increment]
  CartId int [ref: > Carts.CartId]
  ProductId int [ref: > Products.ProductId]
  Quantity int [not null]
}
Table Reviews {
  ReviewId int [pk, increment]
  ProductId int [ref: > Products.ProductId]
  UserId nvarchar(450) [ref: > Users.Id]
  Rating tinyint [not null]
  IsVerifiedPurchase bit [default: 0]
  IsVisible bit [default: 1]
}
Table Appointments {
  AppointmentId int [pk, increment]
  UserId nvarchar(450) [ref: > Users.Id]
  AppointmentDate datetime2 [not null]
  Status nvarchar(20) [default: 'Pending']
}
Table AuditLogs {
  LogId int [pk, increment]
  ActorUserId nvarchar(450) [ref: > Users.Id]
  ActionType nvarchar(50) [not null]
  CreatedAt datetime2 [not null]
}
```
